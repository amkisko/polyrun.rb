# Multi-database example

**`multi_demo`** — Rails app with:

- **Rails multiple databases**: **`primary`** + **`cache`** (separate SQLite files, `db/cache_migrate/`).
- **`CacheRecord` / `CacheEntry`** on the **cache** database.
- **`polyrun.yml`**: **`partition`** (paths, shard index/total), **`prepare`**, **`databases`** for **PostgreSQL** shard DB names when using `bin/polyrun env` in real CI.

This directory is also the **target of `examples/cost_balancing/rails_demo`** (symlink) so cost-based partitioning demos share one Rails tree.

**Lattice suite:** `lib/demo/lattice/` + 120 `spec/demo/lattice/*_spec.rb` files — generated by `examples/script/generate_lattice_spec_suite.rb` (gitignored; invoked from `spec/spec_helper.rb` before `rspec`).

## Polyrun: parallel + merged coverage

`spec/spec_helper.rb` starts **`Polyrun::Coverage::Rails`**. Use **`polyrun parallel-rspec`** (or **`./bin/rspec_parallel`**) so **`run-shards`** and **`merge-coverage`** run in one step:

```bash
cd multi_demo
./script/ci_prepare
RAILS_ENV=test bin/rails db:prepare

bundle exec polyrun -c polyrun.yml parallel-rspec --workers 4
# or: chmod +x bin/rspec_parallel && ./bin/rspec_parallel
```

**Cost-balanced** shards (merged timing JSON): see **`../cost_balancing/demo_rails.sh`** and **`../cost_balancing/polyrun_rails.yml`**. **All strategies** (round-robin, random, cost, HRW) are summarized in **`../partition_strategies/README.md`**.

## Commands

```bash
cd multi_demo
bundle install
./script/ci_prepare
bin/rails db:migrate
bin/rails db:migrate:cache
RAILS_ENV=test bin/rails db:migrate db:migrate:cache
bundle exec rspec
```

See **`../TESTING_REQUIREMENTS.md`** — use **distinct databases per shard** under parallel load.

## Browser system specs

Capybara **system** specs use **Playwright** when the CLI is available; install with **`npm install playwright`** and **`npx playwright install chromium`** (see **`../simple/README.md`**). There is no env flag to force `rack_test` instead.

## Postgres + parallel shards (Docker)

The repo ships **`../docker-compose.yml`** (included by this directory’s **`docker-compose.yml`**) with init SQL that creates **`multi_demo_template`**. From **`examples/`**:

```bash
./script/docker_up.sh
export PGPORT=5433 PGPASSWORD=postgres
./script/docker_polyrun_provision_demo.sh multi_database/multi_demo
cd multi_demo
bundle exec ../../../bin/polyrun -c polyrun.yml parallel-rspec --workers 4
```

Inspect exports with **`bundle exec ../../../bin/polyrun -c polyrun.yml env --shard 0 --total 4`**. For cloud Postgres, create the same template + shard databases and set **`PGPORT`** / **`PGPASSWORD`** as needed.
